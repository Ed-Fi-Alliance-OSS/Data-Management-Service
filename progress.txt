## Task: Prevent MVCC ALTER DATABASE failure due to connection pooling (01-ddl-provision-command)
- Added SqlConnection.ClearPool(connection) after transaction commit in ExecuteInTransaction
  - Root cause: disposed SqlConnection returns to ADO.NET pool, pooled connection blocks ALTER DATABASE exclusive access
- Added WITH ROLLBACK IMMEDIATE to both ALTER DATABASE SET statements in CheckOrConfigureMvcc
  - Defense in depth: forces SQL Server to disconnect any remaining sessions before applying the setting
