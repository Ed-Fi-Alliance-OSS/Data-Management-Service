## Task 2 Complete — Handle missing EffectiveSchema singleton row
Story: 02-preflight-and-idempotency.md

If dms.EffectiveSchema table exists but singleton row (Id=1) is missing, both
PreflightSchemaHashCheck and PreflightSeedValidation now throw InvalidOperationException
with actionable message instead of silently skipping validation. This prevents stale
seed data from a prior partial provisioning from going undetected.

Decision: Fail fast rather than attempt recovery. The table+row are created atomically
in the same DDL transaction, so a missing row indicates partial/corrupt state.
