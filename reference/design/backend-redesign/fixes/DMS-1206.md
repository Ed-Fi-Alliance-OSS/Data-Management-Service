# Bug: Relational model emits XSD-choice reference branches as NOT NULL columns, making SurveyResponse un-insertable

## Description

Found during [DMS-1159](https://edfi.atlassian.net/browse/DMS-1159#icft=DMS-1159) populated-template validation (Grand Bend bulk load, Ed-Fi-Data-Standard v5.2.0, BulkLoadClient 7.3.464, EdFiSandbox claim set, DS52+TPDM relational stack). The defect is on main: the [DMS-1159](https://edfi.atlassian.net/browse/DMS-1159#icft=DMS-1159) branch carries no RelationalModel/Ddl changes.

## Symptom

12x HTTP 500 "Relational write failed for resource 'Ed-Fi.SurveyResponse' because the database reported an unrecognized final write failure." The PostgreSQL log shows the underlying error:

```text
null value in column "SurveyResponderChoiceContact_DocumentId" of relation "SurveyResponse" violates not-null constraint
```

Zero SurveyResponse rows landed in the Grand Bend load - the resource is un-insertable on the relational backend.

## Root Cause

`edfi."SurveyResponse"` materializes all three XSD-choice responder branches as NOT NULL columns (`SurveyResponderChoiceContact_*`, `SurveyResponderChoiceStaff_*`, `SurveyResponderChoiceStudent_*` DocumentId + UniqueId columns). At most one choice member can be present in any document, so the constraints are mutually unsatisfiable. ApiSchema `documentPathsMapping` marks each choice member `isRequired: true` and the relational model trusts it, while `jsonSchemaForInsert` requires only `surveyReference` - which is why the legacy JSONB backend accepted these documents and the defect only surfaces relationally.

SurveyResponse is the only affected table in the DS52+TPDM surface (information_schema scan for NOT NULL choice-branch columns), but the fix must use actual schema/choice semantics, not a column-name substring rule.

## Fix Shape

- Fix in the DMS relational model now; do not wait on upstream ApiSchema/MetaEd. Choice-branch reference columns must be nullable unless the selected branch requires them; mutually exclusive branches cannot all be NOT NULL.
- An upstream ApiSchema/MetaEd correctness follow-up will be filed separately (`documentPathsMapping.isRequired = true` for choice members contradicts `jsonSchemaForInsert`).
- Optional, only if it stays small: map SQLSTATE 23502 (not-null violation) to a clearer error than the generic "unrecognized final write failure" 500; otherwise split into a nonblocking error-mapping hardening ticket.

## Acceptance Criteria

- POST SurveyResponse with exactly one responder choice branch (or none, per the JSON insert schema) succeeds on the relational backend.
- Grand Bend populated bulk load inserts all SurveyResponse rows.
- Blocks [DMS-1159](https://edfi.atlassian.net/browse/DMS-1159#icft=DMS-1159) green populated validation. The `-RequirePopulatedData` restore verification compares source counts; silently absent SurveyResponses are not acceptable.
